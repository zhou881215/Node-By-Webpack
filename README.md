# node 开发方案 1：直接开发，直接部署

- 搭建 node 工程，直接开发
- 开发过程中使用 git 进行管理
- 开发完成后，提交 git
- 进入部署服务器，从 git 中拉取最新代码，然后`npm install`

引发的问题：

- 服务器在`npm install`的过程中，会占用比较大的网络资源
- 代码没有压缩，拉取速度较慢
- 开发过程中，无法使用较新的语法
- 开发过程中，无法使用 ES6 模块化

# node 开发方案 2：直接开发，用 webpack 打包，然后部署

1.  搭建 node + webpack 工程
2.  开发后，使用 webpack 打包
3.  将打包结果上传到服务器，服务器直接运行

### 生产环境的运行：

1. 监控源代码目录，如果源代码有变动
2. 将环境变量设置为`development`，然后进行打包
3. 运行打包结果
